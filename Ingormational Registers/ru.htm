Что имеем?

Хотим хранить информацию о ценах конкурентов на продаваемые нами товары
Хотим отслеживать динамику изменений цен
...а если еще хотим и периодически ее хранить?

Так что же такое Регистр Сведений?

Регистр сведений

Фактически это массив данных, необходимый, чтобы реализовать функцию, которая может выдать необходимую информацию по определенному набору аргументов.

Регистр сведений

  1. Аргументы - измерения
  2. Результат функции - ресурсы
      3. ресурсов может быть больше, чем один (например, оптовая и розничная цена)

А что с периодом?

Периодический регистр

Для разворота информации во времени используется поле Период. Оно не вносится в качестве измерения, а добавляется системой автоматически при создании периодического регистра.

Какие операции поддерживает система, если регистр периодический?

  1. получение наиболее позднего/раннего значения по определенной комбинации измерений
  2. получения среза наиболее поздних/ранних значений

Например

  1. получение последней введенной цены по конкретному товару и конкретному конкуренту
  2. могут быть получены все последние введенные цены по различным товарам и конкурентам

Не периодический регистр

Если регистр не периодический, то поле Период для него не создается. Регистр может быть непериодическим, если не нужно хранить историю изменения цен, а хотим иметь только актуальные цены.

В итоге имеем?

Принцип работы регистра сведений

В системе может быть только одна запись с определенным набором и периодом измерений.

Уникальность регистра сведений

Уникальность записей по набору измерений принципиально отличает регистры сведений от регистров накоплений, которые позволяют вводить несколько записей с одинаковым значением измерений и периодом.

Регистр сведений

  1. Строки регистра сведений - записи
  2. Вносить информацию в регистр сведений можно:
    • вручную
    • документами

Регистр сведений

Документ, которым вносится запись в регистр сведений, называется регистратором.

Если регистр записывается регистратором - записи будут жестко подчинены регистраторам - документам. Записи будут порождаться при проведении документов. При удалении документа записи будут удаляться автоматически.

Регистр сведений в программных модулях

Общие действия над регистром (поиск, выбор и создание записей)

  • объект РегистрСведенийМенеджер.<Имя регистра сведений>

Регистр сведений в программных модулях

Чтения, запись, удаление отдельных записей регистра сведений, не управляемого регистраторами

  • объект РегистрСведенийМенеджерЗаписи.<Имя регистра сведений>

Регистр сведений в программных модулях

Для считывания и занесения набора записей в базу данных по определенному условию отбора

  • объект РегистрСведенийНаборЗаписей.<Имя регистра сведений>

Регистр сведений в программных модулях

Для динамического обхода записей регистра

  • объект РегистрСведенийВыборка.<Имя регистра сведений>

Примеры

  • Перейти к листингу 1
  • Перейти к листингу 2
  • Перейти к листингу 3
